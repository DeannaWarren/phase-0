So what exactly is version control? It's quite simple really. Version control is what developers use to track changes in their code. You may have seen programs listed as 1.0, 1.1, 1.2, and wondered what that means. It simply means what version of the code is being used! Now those changes are on a larger scale, consisting of many small changes, but version control can be used down to a single line of code being changed. This enables the developer to quickly look back through the code and see what was done, when, and even why if good messages are used with each 'save point'. This is where branches become important. You can think of branches very much like the branches on a tree. The main code, usually the working code that is being actively used, is commonly called the master branch, but can really be considered the trunk of our code tree. Each new branch is a copy of the code it is being pulled off of, which is often the master branch, but can also be a branch itself. When changes are made to this code, it doesn't affect the master branch until it is completed and implemented.
